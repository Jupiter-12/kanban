# 项目成员管理

## 变更描述

为看板系统添加权限管理功能，实现：
1. **项目全局可见** - 所有用户登录后能看到所有项目和任务
2. **编辑权限控制** - 基于用户角色控制编辑权限
3. **角色管理** - 所有者可以管理其他用户的角色
4. **用户管理** - 所有者可以增删用户、修改用户信息
5. **个人设置** - 所有用户可以查看和修改自己的个人信息

## 用户角色

| 角色 | 查看项目 | 编辑自己项目 | 编辑所有项目 | 管理用户 | 个人设置 |
|------|----------|-------------|-------------|----------|----------|
| 普通用户 | ✓ | ✓ | ✗ | ✗ | ✓ |
| 管理员 | ✓ | ✓ | ✓ | ✗ | ✓ |
| 所有者 | ✓ | ✓ | ✓ | ✓ | ✓ |

**说明：**
- **所有者 (Owner)** - 超级管理员，拥有所有权限，可以增删用户、修改任何用户信息
- **管理员 (Admin)** - 可以编辑所有项目，但不能管理用户
- **普通用户 (User)** - 只能编辑自己创建的项目

## 功能模块

### 1. 用户管理（仅所有者）
- 查看所有用户列表（用户名、显示名称、邮箱、状态、角色、创建时间）
- 创建新用户
- 编辑用户信息（显示名称、邮箱、密码、账户状态）
- 删除用户（不能删除自己）
- 修改用户角色

### 2. 个人设置（所有用户）
- 查看个人信息
- 修改显示名称
- 修改邮箱
- 修改密码

### 3. 项目分类
- 我的项目：只显示当前用户创建的项目
- 所有项目：显示系统中所有项目

## 影响范围

- **后端**：用户模型（新增 role 字段）、权限检查逻辑、API 端点
- **前端**：项目列表显示、权限判断、角色管理界面、个人设置页面
- **数据库**：users 表新增 role 字段
